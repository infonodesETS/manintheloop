# Man in the Loop — info.nodes Explorer

Standalone web application for exploring the defence supply chain, European Defence Fund landscape, and related data from `database.json` and `edf_calls.json`.
Lives entirely inside `refactoring/` and has no dependency on any other tool in the repository.

> **Development status**: `refactoring/` is the active development branch. It is excluded from the parent repository's git tracking (`.gitignore`). Changes are tracked manually in [`CHANGELOG.md`](./CHANGELOG.md) instead of git commits. When features stabilise they are promoted to the parent (production) repository.
> **Build status**: Under Construction — active development, UI and data subject to change.

---

## Experimental pages

### `automated-investigation.html`

A standalone long-form investigative page — an experiment in generating a narrative report directly from the same data. It uses the same design system as the main explorer (Barlow Condensed, JetBrains Mono, green-accent terminal palette) but presents the data as a scrollable editorial investigation with chapters, charts, a D3 network graph, and company/investor profiles.

**Not linked from `index.html` or any other page.** Access directly via the URL: `/automated-investigation.html`.

---

## How to run

```bash
cd refactoring/
python3 -m http.server 8000
```

Then open `http://localhost:8000` in your browser.

> The app fetches `data/database.json` at runtime. It will not work when opened directly as a `file://` URL due to CORS restrictions on local fetches.

## Tabs

### Supply Chain — Overview
Summary statistics loaded from `database.json`:
- Stat cards: companies, investors, relationships, lead investments, Wikidata coverage
- Wikidata coverage progress bar (% of companies with a QID)
- Sector breakdown bar chart
- Top 8 investors by portfolio size

### Supply Chain — Map
Geographic view of the investment network.
- Country circles on a `d3.geoNaturalEarth1()` projection, sized by company count
- Investment arcs connecting co-invested countries; arc/toggle, reset zoom
- Country click → side panel with company list, sector breakdown, active investors
- Entity filter: click company or investor to highlight connected countries

### Supply Chain — Graph
D3 force-directed graph with three layout modes: Network · Bipartite · Projection. Sector filtering and pan/zoom.

### Supply Chain — Companies
Sortable, filterable table of all company entities (`IN-NNNN`). Click any row to open an entity detail sidebar.
Columns: Name · Sector · Country · Founded · Funding (USD) · Industries · Investor count · Wikidata QID · Validation status

Sidebar sections: CB description (top) · key facts (country, founded) · **Crunchbase block** (amber; profile URL, HQ, website, domain, stage, revenue, funding, CB rank, investor type, patents, acquired by, founders, industries, industry groups) · **Wikidata block** (teal; description, aliases, type, HQ, employees, ISIN, official site, Wikipedia/Wikidata links) · focus · investors · validation issues · both blocks show "not available" when data is absent

### Supply Chain — Investors
Sortable, filterable table of all investor entities (`IV-NNNN`). Click any row to open an entity detail sidebar.
Columns: Name · Type · Portfolio size · Lead count · Portfolio sample

Sidebar sections: CB description (top, if present) · portfolio stats · **Crunchbase block** · **Wikidata block** (131/242 investors have Wikidata data; 111 show "not available") · portfolio company list

### Supply Chain — Relationships
Flat table of all `REL-NNNN` edges. Click an investor name or company name cell to open its entity detail sidebar.
Columns: ID · Investor · Company · Sector · Role (lead / follow)

### Supply Chain — Matrix
Investment matrix: investors as rows, companies as columns. Filter by sector; click cell / header for detail panel.

### European Defence Fund — Map
Geographic view of EDF-funded participants, sourced from `data/edf_calls.json`.
- Country circles on a `d3.geoNaturalEarth1()` projection, sized by organisation count (√ scale, range 3–12 px)
- MultiPolygon centroid correction: for countries with overseas territories (France, Spain, Portugal, Norway…) the centroid is computed on the largest polygon by `d3.geoArea()`, keeping the marker on the mainland
- Connection arcs between countries that share at least one funded project; arc opacity scales with co-project count
- **Country click**: dims unrelated countries/nodes, highlights the selected cluster, and shows only arcs connected to that country; sidebar lists all beneficiary organisations for that country
- **Org click** (from sidebar): shows only arcs from that organisation's country to its partner countries; sidebar lists all projects the organisation participated in, with role badge, acronym, EU contribution, and EC Portal link
- **Reset** button clears all filters and restores all arcs
- Arc toggle to show/hide all arcs; reset zoom button
- Labels are fixed at 16 px screen size regardless of zoom level (inline `font-size = 16/k` SVG units)
- Initial view centred on Europe (`translate(-2926, -261) scale(3.647)`); live tx/ty/k position display for calibration
- Green colour scheme matching Supply Chain Map: `var(--accent)` nodes, teal arcs (`rgba(104,204,209,0.6)`)

### European Defence Fund — Overview
On-the-fly metrics computed from `data/edf_calls.json` every page load.
Two-row stat grid (4 columns):
- Row 1: European Defence Fund Calls · Total Allocated Budget (topic-deduped, ~€5.07B for 120/201 calls)
- Row 2: Calls with Funded Projects · Total EU Contribution (from participant records, matches Beneficiaries) · Funded Projects · Unique Participants
- Top-20 countries by participation count (with EU contribution if available)
- Top-20 organisations by project count (with EU contribution if available)
- Graceful "no data yet" placeholder when `projects` arrays are empty

### European Defence Fund — EDF Calls Search
Search and explore EDF calls and their funded projects.
- Autocomplete on all 201 calls; fast path (cached JSON) + live EU API fallback
- Participant sidebar on click

### European Defence Fund — EDF Beneficiaries
Participant-focused explorer aggregating all funded project participations by organisation.
- Toolbar + stats bar in single header row; country filter (A-Z), funded-only toggle, sortable table
- Click any row → slide-in org detail sidebar (projects list with role, funding, dates)
- 50 rows per page with pagination

### Tools — Wikidata Inspector
Per-company Wikidata lookup. Local mode (browse `database.json`) and live search mode (full Wikidata autocomplete).
Six parallel SPARQL queries: identity, key people, corporate structure, web links, stock data, financial history.

### Tools — Data Quality
Audit view: QID coverage bar, validation status breakdown, missing QID list, open validation issues.

### Tools — Known Issues
Renders `known-issues.md` via `marked.js`. Documents in-app data and calculation issues with status tags.

## Data model (summary)

The app reads a single file: `data/database.json` (schema v2.0).

```json
{
  "_schema": "2.0",
  "_updated": "YYYY-MM-DD",
  "entities": [ ... ],
  "relationships": [ ... ]
}
```

### Entity IDs
| Prefix | Type |
|---|---|
| `IN-NNNN` | Companies (manufacturers, tracked entities) |
| `IV-NNNN` | Investors (funds, banks, government agencies, institutions) |

### Relationship IDs
| Prefix | Type |
|---|---|
| `REL-NNNN` | Investment relationship (source = investor, target = company) |

See `SCHEMA.md` for the full field specification and `UPDATE_PROTOCOL.md` for how to reconcile new data against the existing graph.
