# Spec — database.json v2.0 Explorer (current version)

_Document date: 2026-03-15. Describes the application as it stands today, before any refactoring._

---

## 1. Purpose

Single-file, single-page web application for exploring and validating the investment-network dataset stored in `data/database.json` (schema v2.0). The app has no build step, no npm dependencies, and runs from `python3 -m http.server`.

## 5. Data Model

### 5.1 `database.json` Top Level

```json
{
  "_schema": "2.0",
  "_updated": "YYYY-MM-DD",
  "entities": [ ...Entity ],
  "relationships": [ ...Relationship ]
}
```

```json

  "id":          "IN-0001" | "IV-0001",
  "type":        "company" | "fund" | "bank" | "government_agency" | "institution",
  "name":        "string",
  "sector":      "Defence" | "Mining" | "Tech" | "Startup",   // companies only
  "roles":       ["investor"],                                  // optional dual-role flag
  "wikidata_id": "Q12345" | null,
  "validation":  [{ "status": "confirmed"|"flagged"|"needs_review", "description": "…", "datestamp": "…" }],
  "sources": {
    "crunchbase": {
      "website":           "https://…",
      "description":       "…",
      "total_funding_usd": 1234567890,
      "industries":        ["Industry A", "Industry B"],
      "headquarters":      "City, Country"
    },
    "wikidata": {
      "label":            "…",
      "description":      "…",
      "country":          "United States",
      "inception":        "1993-01-01",
      "employees":        50000,
      "headquarters":     "…",
      "official_website": "https://…",
      "isin":             "…",
      "wikipedia_url":    "https://en.wikipedia.org/wiki/…"
    },
    "infonodes": {
      "country":        "…",
      "wikipedia_url":  "…"    // WARNING: may contain legacy cross-contamination errors
    }
  }
}
```

### 5.3 Relationship Shape

```json
{
  "id":     "REL-0001",
  "type":   "investment",
  "source": "IV-0001",    // investor entity ID
  "target": "IN-0001",    // company entity ID
  "details": {
    "lead": true | false
  }
}
```

### 5.4 Entity ID Prefixes

| Prefix | Entity type |
|---|---|
| `IN-NNNN` | Company (manufacturer / tracked entity) |
| `IV-NNNN` | Investor (fund / bank / government_agency / institution) |
| `REL-NNNN` | Investment relationship |

## 6. Application State (`state.js`)

A single exported object `AppState`. No immutability, no getters/setters — all properties are directly mutable.

```javascript
AppState = {
  // Raw db
  db:            null,        // full parsed database.json

  // Entity arrays (populated by loadData)
  companies:     [],          // all entities where type === 'company'
  investors:     [],          // all entities where type !== 'company'
  relationships: [],          // all relationships where type === 'investment'

  // Filtered working copies used by matrix & graph
  ALL_COMPANIES: [],          // immutable reference copy of companies
  COMPANIES:     [],          // mutable — filtered by current sector
  INVESTORS:     [],          // sorted by total desc; entries are investorMeta objects

  // Derived lookups (populated by loadData, never mutated after)
  derived: {
    entityMap:    {},         // id → entity
    investorMeta: {},         // investor id → { entity, portfolio, leads, total }
    raw:          [],         // [companyName, investorName, isLead] tuples
    companyMap:   {},         // company name → entity
    invMap:       {},         // investor name → investorMeta
  },

  // Per-tab UI state
  ui: {
    currentTab: 'overview',

    matrix: {
      sector: 'all'
    },
    companies: {
      sector: 'all',
      sort:   { key: 'name', asc: true },
      search: ''              // not actually stored here — read from DOM input
    },
    investors: {
      sort:   { key: 'total', asc: false },
      search: ''              // not actually stored here — read from DOM input
    },
    relationships: {
      search: ''              // not actually stored here — read from DOM input
    },
    graph: {
      view:       'network',  // 'network' | 'bipartite' | 'projection'
      sector:     'all',
      projFilter: 'all',      // 'all' | 'multi'
      sim:        null,       // D3 force simulation (network)
      simBi:      null,       // D3 force simulation (bipartite)
      simProj:    null,       // D3 force simulation (projection)
    },
    wikidata: {
      selectedCountry: null,
      liveMode:        false,
      list:            [],    // filtered companies array
      debounce:        null,  // setTimeout handle
    },
    eucalls: {
      built: false            // lazy-init guard
    },
    map: {
      built:       false,     // lazy-init guard
      activeFilter: null,     // { id, name, isos: Set } | null
      showArcs:    true,
      zoom:        null,      // D3 zoom behaviour reference
      svg:         null,      // D3 selection of #map-svg
      g:           null,      // D3 selection of root <g>

      countryData: {},        // iso → { name, companies[] }
      arcData:     [],        // [{ source_iso, target_iso, weight }]
      centroids:   {},        // iso → [x, y] in projected coordinates
    },
  },
};
```

## 7. Data Loading (`data.js`)

`loadData()` — called once on boot, returns the raw `db` object. All derived structures are built here and written directly into `AppState`.

**Steps:**
1. `fetch('data/database.json')` → parse JSON
2. Populate `AppState.derived.entityMap` (id → entity)
3. Segregate `AppState.companies` and `AppState.investors` by `type`
4. Filter `AppState.relationships` to `type === 'investment'`
5. Build `AppState.derived.investorMeta`:
   - Key: investor entity ID
   - Value: `{ entity, portfolio: [{ company, lead }], total, leads }`
6. Build company → investors lookup (`c._investors = [{ name, lead }]`) — written as a direct property on each company entity
7. Build `AppState.derived.raw` — `[companyName, investorName, isLead]` tuples for matrix and graph
8. Build `AppState.derived.companyMap` (name → entity)
9. Build `AppState.derived.invMap` (name → investorMeta)
10. Set `AppState.ALL_COMPANIES` and `AppState.COMPANIES` as copies of `AppState.companies`
11. Set `AppState.INVESTORS` sorted by `total desc`

## 8. Bootstrap Flow (`main.js`)

```
loadData()
  └─ populates AppState
  └─ sets nav-meta (updated date) and tnav-info (counts)
  └─ calls init functions:
       initOverview()        → renders stats immediately
       initMatrix()          → renders matrix immediately
       initCompanies()       → wires search, renders table
```

### 8.2 `showTab(name)`

```
1. Read AppState.ui.currentTab (prev tab)
2. Toggle .active class on all .tnav-btn and .tab-pane
3. Set AppState.ui.currentTab = name
4. If prev === 'graph' and name !== 'graph': pauseGraph()
5. If name === 'graph' and sim exists: resumeGraph()
6. Lazy init on first visit:
   - graph:   call initGraph() when AppState.ui.graph.sim is null
   - map:     call initMap()   when AppState.ui.map.built is false
   - eucalls: call initEucalls() when AppState.ui.eucalls.built is false
              set AppState.ui.eucalls.built = true
```

**Table structure (rendered into `#matrix-scroll`):**

- Header row: `INVESTOR` label column | one `<th class="col-hdr">` per company | `TOTAL` column

  - Company cells: `.dot-lead` or `.dot-follow` dot if invested, `.dot-none` otherwise
  - Total cell: mini bar + count number

**Color coding for count pill:**
- `c3` = 3+ investments (accent color)
- `c2` = 2 investments (accent-dim)
- `c1` = 1 investment (white 35%)

**Detail panel — investor fields:**
- Name, type badge, investment counts
- Portfolio list (sorted leads first, filtered to current sector)
- Co-investors list (up to 6, with co-invest count)

**Detail panel — company fields:**
- Name, sector badge, manufacturer badge, dual badge if applicable
- Country + inception from `sources.wikidata`
- Funding from `sources.crunchbase`
- Description (truncated to 200 chars) from crunchbase or wikidata
- Website + Wikipedia links
- Investor list (sorted leads first)

**Tooltip:** investment dots show hover tooltip via `data-action="tip"` + `data-tip-*` attributes.

### 10.6 Graph (`tabs/graph.js`)

**Lazy init:** `initGraph()` called by `showTab('graph')` only when `AppState.ui.graph.sim` is null. Calls `buildGraphView()`.

**Exports:**
- `setGraphView(v)` — switches layout mode, updates button active states, shows/hides projection filter buttons, calls `buildGraphView()`
- `setGraphSector(s)` — filters `AppState.COMPANIES`, calls `buildGraphView()`
- `setProjFilter(f)` — sets `AppState.ui.graph.projFilter`, calls `buildGraphView()`
- `closeGraphDetail()` — removes `.open` from `#graph-detail`
- `pauseGraph()` — stops all three simulations
- `resumeGraph()` — restarts all three simulations

**Layout DOM:**

```
#tab-graph
  └─ #graph-layout
       ├─ #graph-pane
       │    ├─ #graph-controls
       │    │    ├─ .gv-btn  — Network / Bipartite / Projection
       │    │    ├─ .sf-btn  — sector filter (All / Defence / Mining / Tech / Startup)
       │    │    └─ #proj-filter-btns  — All edges / Co-invest ≥2 (hidden except in Projection)
       │    └─ #graph-svg   — D3 SVG canvas
       └─ #graph-detail     — slide-in right detail panel
            └─ #graph-inner — panel content
```

**Three layout modes:**

| Mode | Key | Description |
|---|---|---|
| Network | `network` | Radial: companies on inner ring (radius 15% of min dimension), investors on outer ring (42%). D3 force with link, charge, radial, collide. Stored in `AppState.ui.graph.sim`. |
| Bipartite | `bipartite` | Two columns: investors at x=25%, companies at x=72%. D3 force with forceX/forceY anchors. Stored in `AppState.ui.graph.simBi`. |
| Projection | `projection` | Investors only. Two investors connected if they share ≥1 portfolio company. Edge weight = count of shared companies. Filter: all edges or only weight > 1. Stored in `AppState.ui.graph.simProj`. |

**Node shapes:**
- Companies → `<circle r=gR_co(d)>` where `gR_co = 4 + sqrt(investors.length) * 2`
- Investors → `<rect>` where `gR_inv = 5 + sqrt(total) * 4`
- Dual-role companies → `<text class="gstar">★` overlay
- All nodes → `<text class="glabel">` label (truncated at 15 chars)
- Investors with portfolio > 1 → `<text class="gbadge gbadge--{typeKey}">` count badge

**Node CSS classes:**
- Companies: `gnode-co gnode-co--{sector}`
- Investors: `gnode-inv gnode-inv--{type}` where type one of `fund | bank | inst | govt`

**Interaction:**
- Drag: D3 drag, `fx`/`fy` pin on drag start, release on end
- Click node: highlight connected subgraph (`.ghl` class), open detail panel
- Click SVG background: clear highlight, close detail panel
- Hover: `tip()` with name + investment counts
- Pan/zoom: D3 zoom with `scaleExtent([0.05, 5])`

| Mode | Forces |

### 10.7 Map (`tabs/map.js`)

**Lazy init:** `initMap()` called by `showTab('map')` only when `AppState.ui.map.built` is false. Sets `.built = true`, then calls `buildMapView()`.

**Exports:**
- `toggleMapArcs(show)` — sets `AppState.ui.map.showArcs`, sets `#map-arc-layer` display
- `resetMapZoom()` — transitions SVG zoom to `zoomIdentity`
- `closeMapPanel()` — adds `d-none` to `#map-panel`, removes `.selected` from countries
- `clearMapFilter()` — sets `AppState.ui.map.activeFilter = null`, calls `applyMapFilter()`

**Country-to-ISO mapping:** hardcoded `WD_TO_ISO` object in module scope (80+ countries, Wikidata label → ISO numeric). Also derives `ISO_TO_NAME` as inverse.

**Data preparation (in `buildMapView()`):**
1. Build `countryData` (iso → `{ name, companies[] }`) from company `sources.infonodes.country` or `sources.wikidata.country`
2. Build `invCountries` (investor id → Set of ISOs of their portfolio companies' countries)
3. Build `pairWeight` (ISO pair key → investor count) — for arc thickness
4. Build `arcData` array (`[{ source_iso, target_iso, weight }]`)
5. Fetch world-atlas TopoJSON from CDN → call `drawMap(world)`

**`drawMap(world)` internals:**
- Projection: `d3.geoNaturalEarth1()`, scale `W/6.2`, centered
- Graticule: thin `#1a1a1a` lines
- Countries: `.map-country` paths; countries with data get `.has-data`
- Arc layer: `<g id="map-arc-layer">` — quadratic Bézier curves, stroke scaled by weight
- Node layer: `<g id="map-node-layer">` — `<circle class="map-node">` sized by `sqrt` scale [4,22]
- Labels: shown only for nodes with `r >= 8`
- Zoom: `scaleExtent([0.5, 12])` — zoom handler compensates node/label/arc sizes inversely to `k` using `datum().baseR` / `datum().baseSw` stored as JavaScript datum properties

**`showMapCountry(iso)` — side panel content:**
- Country title in `#map-panel-title`
- `#map-panel-body` innerHTML: companies list + sector breakdown + active investors list (capped at 25)
- Each company and investor item has `data-action="filterMapByEntity"` + `data-id`
- After render, event listeners are re-attached to those items

**`filterMapByEntity(entityId)`:**
- If `IN-` (company): adds own country + all countries where any shared investor also operates
- If `IV-` (investor): adds all countries of portfolio companies
- Sets `AppState.ui.map.activeFilter = { id, name, isos: Set }`
- Calls `applyMapFilter()`

**`applyMapFilter()`:**
- Shows/hides `#map-filter-bar`
- Applies `.country-dim` / `.node-dim` / `.node-focus` / `.arc-dim` CSS classes based on whether ISO is in `activeFilter.isos`

```
       │    ├─ #map-filter-bar   — active filter indicator + clear btn
```

### 10.8 Wikidata Inspector (`tabs/wikidata.js`)

**Init:** `initWikidata()` → calls `renderWdSidebar()`, wires outside-click to close `#wd-live-drop`.

**Module-level state (not in AppState):**
- `wdCache: Map<QID, html>` — in-memory cache of rendered SPARQL results
- `wdAbortCtrl: AbortController | null` — cancels in-flight SPARQL request on new selection

**Exports:**
- `toggleWdMode(isLive)` — switches between local-browse and live-search mode
- `onLiveInput()` — debounced (300ms) trigger for `fetchWdAutocomplete()`

**Two modes:**

**Local mode (default):**
1. `renderWdSidebar()` — builds country filter buttons from company Wikidata countries; calls `renderWdList()`
2. `renderWdList()` — filters to selected country (or all), renders `#wd-company-list` items
3. Click item → `selectWdCompany(qid, label)` → `fetchWdData(qid)`

**Live mode:**
1. `#wd-live-input` input → `onLiveInput()` → debounce → `fetchWdAutocomplete(q)` → Wikidata `wbsearchentities` API
2. Dropdown `#wd-live-drop` populated with results
3. Click item → `selectWdLive(qid, label)` → `fetchWdData(qid)`

**`fetchWdData(qid)`:**
- Checks `wdCache` first
- Cancels previous request via `AbortController`
- Fires 6 parallel SPARQL queries via `Promise.all()`:
  1. `qCore(id)` — label, description, country, inception, legal form, founders, sectors, HQ, LEI, Wikipedia URL
  2. `qPeople(id)` — CEO history, ownership history, board members
  3. `qCorporate(id)` — parent organizations, subsidiaries, products/services
  4. `qSocial(id)` — website, logo, social handles (Twitter, LinkedIn, Facebook, Instagram, GitHub, Crunchbase, Bloomberg)
  5. `qStock(id)` — stock exchanges, ticker symbols, ISIN codes, SEC CIK, SWIFT/BIC
  6. `qFin(id)` — financial history (Market Cap, Revenue, Net Income, Operating Income, Assets, Equity, Liabilities, Debt, Employees) per year
- Merges result bindings from queries 1–5 into single object
- Calls `renderWdResults()` → stores HTML in `wdCache` → injects to `#wd-results`

**`renderWdResults(result, qid)` output:**
- Optional "Stored in database.json" table showing `sources.wikidata` fields
- "Live from Wikidata SPARQL" table with 34 possible fields (label → value mapping)
- Image fields rendered as `<img>` if URL matches image extension
- URL fields rendered as `<a target="_blank">`
- Multi-value fields split by `"; "` separator
- Financial history rendered via `renderFinTable()` — nested `<table>` by year × metric

**SPARQL endpoint:** `https://query.wikidata.org/sparql`

**Layout DOM:**

```

  └─ #wd-layout
       ├─ #wd-sidebar
       │    ├─ #wd-sidebar-top
       │    │    ├─ #wd-mode-toggle    — live/local switch
       │    │    ├─ #wd-local-controls — country filter buttons (#wd-country-btns)
       │    │    └─ #wd-live-controls  — search input (#wd-live-input) + dropdown (#wd-live-drop)
```

**Lazy init:** `initEucalls()` called by `showTab('eucalls')` once. Sets `AppState.ui.eucalls.built = true`.

**All DOM IDs are prefixed `ec-`** to avoid namespace collisions.

**Module-level state (not in AppState):**

```javascript

currentPattern   // string | null
currentYears     // string[]
edfCallsList     // [{ identifier, title, status, deadline }] — all calls for autocomplete
```

**Configuration constants:**

```
1. Try static file: fetch('data/edf_calls.json')
   → success: populate edfCallsList + edfCallsDb; trigger background deltaCheck()
2. Fall back: check localStorage['edf_calls_v1'] (with TTL)
3. Fall back: fetch live from EU Search API
   → iterate years, paginate with PAGE_SIZE=50
   → progressive autocomplete updates every page
```

**`deltaCheck()`:** one background API call on tab open to detect new calls not in the cached file.

**CORS proxy chain (`proxiedFetch`):**

```
GET requests:  direct → allorigins.win → corsproxy.io
POST requests: direct → corsproxy.io
```

**Search flow:**

```

  → autocomplete dropdown (#ec-callDrop) appears with matching calls
  → user selects or presses Enter
  → if not EDF- prefix: show Bootstrap modal (#ec-edfOnlyModal)
  → setCallStatus('fetching')
  → show #ec-progressSection
  → for each year in YEARS_TO_TRY:
       if identifier found in edfCallsDb: use cached data (fast path)
       else: call EU SEARCH_API for that year
  → fetch projects from PROJECTS_API for each found identifier
  → aggregate results
  → render results:
```

## 14. Rendering Approach

All tab renders use `innerHTML` assignment with string concatenation:

```javascript
element.innerHTML = list.map(item => `<tr>...</tr>`).join('');
```

XSS protection is applied through the `esc()` helper, which must be called manually on every user-derived or data-derived string inserted into HTML templates.

There is no virtual DOM, no diffing, and no DOM reconciliation. Every filter or sort operation replaces the entire DOM subtree of the target container.

---

## 15. Module System

All JavaScript files use `'use strict'` and ES module syntax (`import` / `export`). `index.html` loads `js/main.js` as `type="module"`. D3, TopoJSON, and Bootstrap are loaded as classic `<script>` globals before the module entry point.

---

## 16. Known Constraints and Behaviors

- **No URL routing.** Tab state is not persisted in the URL. Refreshing always opens to Overview.
- **No debounce** on search inputs in Companies, Investors, or Relationships tabs. Every keystroke triggers a full re-render.
- **Graph simulations** accumulate in `AppState.ui.graph` and are paused/resumed but never garbage-collected. Switching view modes stops the old simulation and creates a new one.
- **Map state** (`svg`, `g`, `projection`, `centroids`) lives in `AppState.ui.map` alongside display state (`showArcs`, `activeFilter`), making the UI state difficult to reset.
- **Eucalls local state** (all 7 module-level variables) is not in AppState and cannot be inspected or reset without reloading the tab.
- **Wikidata cache** (`wdCache`) is a module-scoped `Map` and persists for the session.
- **Country lookup** in the Map tab uses a hardcoded `WD_TO_ISO` table; countries not in the table are silently ignored.
