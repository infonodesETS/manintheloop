## CSV format notes

**English-locale CSVs (PICK, IXN, IXP):**
- Header detected by: line starting with `Ticker,`
- Number format: `1,234,567.89` (comma thousands, dot decimal)
- Asset class filter: `Asset Class == "Equity"`
- Parsed by: `scripts/parse_ishares.py` (no conversion needed)

**Italian-locale CSV (DFND original):**
- Number format: `1.234.567,89` (dot thousands, comma decimal)
- Column headers in Italian (e.g. `Ponderazione (%)`, `Area Geografica`)
- Asset class value: `"Azionario"` instead of `"Equity"`
- Requires conversion before parsing — see `rawdata/ishare_aerospace_defense_GICS201010_en.csv`
- Conversion logic: inline Python script run 2026-04-24 (not saved as a standalone script)

## Adding a new ETF — checklist

1. Save raw CSV to `rawdata/` with naming pattern `ishares_<sector>_gics<code>[_en].csv`
2. If Italian locale: run conversion (see DFND section above for column mapping + number format)
3. Write `scripts/import_ishares_<sector>.py` following `import_ishares_aerospace.py` pattern
4. Add ETF metadata to this file
5. Update `scripts/import_ishares_<sector>.py` constants (ETF_NAME, ETF_TICKER, GICS_CODE, EXTRACTED_AT)
6. Run `--dry-run`, review matches and net-new entities
7. Apply, validate (`python3 scripts/validate.py`), update STATUS.md, commit
